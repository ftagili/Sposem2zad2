# SPO: задание 2, вариант 18

Реализация второй лабораторной работы для варианта `18`.

По методичке вариант `18` означает не SQL-вариант `18`, а две выборки из раздела DBIS lab #3:

- `48` — синхронный режим;
- `78` — неблокирующий режим.

Текущая папка содержит и рабочую C-реализацию, и отдельные материалы для замечаний преподавателя: текст программы на языке из первого семестра и SQL-проверку результатов.

Важно для защиты:

- `spo_task2_v18.c` — это не “посторонний C-код”, а как раз реализация runtime-части: pipe, чтение/запись, синхронизация, планирование и остальная исполняющая логика.
- `spo_task2_v18_pipeline.src` + `spo_task2_v18_workers.src` — это полный текст программы на вашем языке: в первом файле основные процедуры `Run48/Run78` и `main`, во втором — сами worker-процедуры `Proc`.
- Преподавателю удобнее показывать эти два `.src` файла вместе, а `spo_task2_v18.c` открывать отдельно уже как реализацию pipe/read/write и синхронизации.

## Что лежит в каталоге

- `spo_task2_v18.c` — runtime, загрузка CSV, pipeline-запросы `48.x` и `78.x`, self-test.
- `spo_task2_v18_pipeline.src` — переписанная основная логика на языке из первого семестра: открытие потоков, `createPipe`, `createThread`, запуск обработчиков и ожидание завершения.
- `spo_task2_v18_workers.src` — тот же язык, но уже для самих процедур-обработчиков: source/filter/join/group/sink.
- `data/` — учебный набор CSV-таблиц.
- `sql_verification.sql` — эквивалентные `SELECT ... JOIN ...` запросы для тех же результатов.
- `verify_sql_results.py` — импортирует CSV в SQLite и сравнивает SQL-результаты с ожидаемыми строками из C-программы.
- `sql_verification_report.md` — готовая сводка по результатам запросов `48.x` и `78.x`.
- `workload_s390x.S` — asm-хелперы для `s390x`.
- `workload_portable.c` — portable fallback.

## Сборка

На Linux / `s390x`:

```bash
make
```

На текущей Windows-машине локально проверяется только stub:

```bash
gcc -std=c11 -O2 -Wall -Wextra -pedantic spo_task2_v18.c workload_portable.c -o spo_task2_v18.exe
```

## Запуск

Все выборки варианта `18`:

```bash
./spo_task2_v18
```

Только `48.x`:

```bash
./spo_task2_v18 --variant 48
```

Только `78.x`:

```bash
./spo_task2_v18 --variant 78
```

Self-test:

```bash
./spo_task2_v18 --self-test
```

## Проверка SQL

Проверка импортирует CSV в SQLite и сверяет результаты со строками, которые ожидает `spo_task2_v18.c`:

```bash
python verify_sql_results.py
```

## Что показать на защите

- `spo_task2_v18_pipeline.src` как основную часть текста программы на вашем языке.
- `spo_task2_v18_workers.src` как продолжение того же текста программы, где лежат реализации процедур обработки.
- `spo_task2_v18.c` как реализацию того, что преподаватель отдельно просил посмотреть: `createPipe`, чтение, запись, синхронизацию и пассивное ожидание.
- `sql_verification.sql` как набор соответствующих `SELECT ... JOIN ...`.
- `sql_verification_report.md` или вывод `python verify_sql_results.py` как подтверждение, что SQL даёт тот же набор данных.

## Примечания

- Для `78.1` результат специально пустой: фильтр `person_id = 153285` и `person_id = 163249` взаимоисключающий.
